springboot自动配置

发表于 2020-03-18

springboot自动装配的流程大致为

  1. 获取@SpringBootApplication注解
  2. 获取AutoConfigurationImportSelector的实例selectImports方法载入配置

@SpringBootApplication

SpringApplication.run需要传入的是一个被@SpringBootApplication修饰的class

Springboot会根据class获取class上修饰的@SpringBootApplication

@SpringBootApplication包含了@EnableAutoConfiguration

@EnableAutoConfiguration通过@Import将AutoConfigurationImportSelector导入了Spring的IOC容器

AutoConfigurationImportSelector的selectImports方法加载过滤配置

getCandidateConfigurations会调用SpringFactoriesLoader.loadFactoryNames

加载classpath下的META-INF/spring.factories文件

spring.factories文件位于org.springframework.boot:spring-boot-autoconfigure:2.0.0.RELEASE包下

org.springframework.boot.autoconfigure.EnableAutoConfiguration键记录了默认会自动装配的配置类

springboot是如何获取AutoConfigurationImportSelector的实例的

sourceClass是一开始在SpringApplication.run传入的DemoApplication.class

collectImports是一个递归函数，会把class以及class/annotation上修饰的annotation都存放到Set里

  • Set<SourceClass> imports ：存储使用@Import导入的class
  • Set<SourceClass> visited：存储其他的class和annotation

DemoApplication.class被@SpringBootApplication修饰

@SpringBootApplication被@EnableAutoConfiguration修饰

@EnableAutoConfiguration使用@Import导入了AutoConfigurationImportSelector.class

因此getImports的结果包含了AutoConfigurationImportSelector.class

Class.forName与ClassLoader loadClass的区别

发表于 2020-03-17

执行区别

Class的forName和ClassLoader的loadClass方法都可以加载类

Class.forName会执行类的静态代码块

ClassLoader.getSystemClassLoader().loadClass不会执行类的静态代码块

由于Class.forName会执行静态代码块的特性，常见的应用有用于注册数据库驱动

通过Class.forName(“com.mysql.cj.jdbc.Driver”)就能直接完成驱动的注册

源码分析

通过源码分析造成区别的原因

Class方式

Class.forName调用的是native修饰的forName0本地方法来执行类的加载

Class.forName(String, boolean, ClassLoader)方法上有关于参数的详细注释

  • name：完全限定名（包名 + 类名）

  • initialize：是否将类初始化，详情可以查看The Java Language Specification的12.4章节

  • loader：类加载器，Class.forName是通过ClassLoader来加载类的

类的初始化阶段

类在初始化时会初始化静态成员变量和执行静态代码块

深入理解Java虚拟机 第二版 周志明

类从被加载到虚拟机内存中开始，到卸载出内存为止，它的整个生命周期包括：加载（Loading）、验证（Verification）、准备（Preparation）、解析（Resolution）、初始化（Initialization）、使用（Using）和卸载（Unloading）7个阶段。其中验证、准备、解析3个部分统称为连接（Linking）

初始化阶段是执行类构造器<clinit>()方法的过程

<clinit>()方法是由编译器自动收集类中的所有类变量的赋值动作和静态语句块（static{}块）中的语句合并产生的

The Java Language Specification 12.4 Oracel官网

12.4. Initialization of Classes and Interfaces

Initialization of a class consists of executing its static initializers and the initializers for static fields (class variables) declared in the class.

Initialization of an interface consists of executing the initializers for fields (constants) declared in the interface.

因此使用Class.forName加载类会使用类加载器加载类并默认初始化类，通过initialize设置是否初始化类

ClassLoader方式

ClassLoader加载类并不会对类进行初始化，因此不会执行静态代码块

ClassLoader的loadClass加载目标类的顺序为

  1. 在已经加载过的类中查找要加载的目标类

  2. 通过双亲委托的方式加载目标类

  3. 通过findClass加载目标类

总结

Class.forName

  • 加载类默认会执行静态代码块

  • 通过initialize参数设置是否初始化类

ClassLoader loadClass

  • 正常加载类，但不会初始化类

类在被初始化时会初始化静态成员变量和执行静态代码块

Class与ClassLoader的getResource方法的区别

发表于 2020-03-10

路径差异

Class和ClassLoader都提供了两种方法来加载资源

  • getResource(String): URL
  • getResourceAsStream(String): InputStream

使用Class与ClassLoader的区别在于填写相同的路径会得到不同的结果

Class方式

ClassLoader方式

源码分析

通过源码分析造成差异的原因

Class和ClassLoader使用的都是ClassLoader的方法去加载资源

但Class使用resolveName对传入的参数做了特殊处理，从而导致了最终结果的不一致

Class对填写的路径做了解析处理

  • 对于以/开头的值去除/
  • 对于不以/开头的值会添加当前类所在包的包名前缀

总结

Class加载资源

  • 填写的是相对路径：得到classpath路径 + 类所在的包路径 + 填写的路径

  • 填写的是绝对路径：得到classpath路径 + 填写的路径

ClassLoader加载资源

  • 填写的是相对路径：得到classpath路径 + 填写的路径
  • 填写的是绝对路径：null

Class、ClassLoader加载资源调的都是ClassLoader的getResource/getResourceAsStream方法

Class加载资源之前会对填写的路径做特殊处理，/开头去除/，否则添加类所在包的包名前缀

Class去除了/开头的路径的/，因此调用ClassLoader的方法不会返回null

mysql group by时不group by null

发表于 2019-11-07

MySQL取消对值为NULL的字段的分组

假定我们现在有一张score表

我们对score表根据grade字段进行分组

可以看到grade字段为NULL的所有数据都被归为一组了

如果这个时候想让grade字段为NULL的数据成为单独的一组，可以使用IFNULL搭配UUID实现

  • IFNULL函数会判断第一个参数是否为NULL，如果为NULL返回第二个参数的值，否则返回第一个参数的值
  • UUID函数会生成一串唯一不重复的UUID

UUID函数生成了唯一不重复的值通过IFNULL函数替代了原本的NULL进行分组

所以分组后的grade字段为NULL的数据都可以单独归为一组

1055错误

当进行分组时可能会发生1055错误

问题原因

这是因为配置了sql_mode=only_full_group_by

这样对于GROUP BY聚合操作，如果SELECT中的列没有出现在GROUP BY中，则这个SQL就是不合法的

查看配置

不同的mysql版本的sql_mode默认配置不同，可以通过以下命令查看配置

  • SELECT @@global.sql_mode;

  • SHOW VARIABLES LIKE 'sql_mode';

MySQL5.6默认没有配置only_full_group_by

MySQL5.7默认配置了only_full_group_by

修改配置

修改MySQL的配置文件，在[mysqld]下配置sql_mode时取消ONLY_FULL_GROUP_BY配置即可

lombok

发表于 2019-11-06

@Accessors

通过该注解可以控制getter、setter方法生成的格式，需要配合@Data或@Getter、@Setter使用

可以在类或成员变量上使用，且在类上使用将应用于所有的成员变量

@Accessors(fluent = true)

生成的getter、setter方法的方法名为属性名，且setter方法返回当前对象

查看反编译后的代码

@Accessors(chain = true)

生成的getter、setter方法名为带get、set前缀的方法名，且setter方法返回当前对象

chain默认为false，但当设置了fluent=true且未设置chain的值时，chain会默认为true

查看反编译后的代码

@Accessors(prefix = “a”)

生成的getter、setter方法名为带get、set前缀的方法名，但方法名不包含a前缀
