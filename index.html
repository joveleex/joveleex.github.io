Class与ClassLoader的getResource的路径差异

发表于 2020-03-10

路径差异

Class和ClassLoader都提供了两种方法来加载资源

  • getResource(String): URL
  • getResourceAsStream(String): InputStream

但使用Class加载资源和使用ClassLoader加载资源时填写相同的路径却会得到不同的结果

如下以使用getResource加载各种目录为例比较两者差异

ClassLoader方式

源码分析

仅通过源码分析造成差异的原因，不探究加载资源的具体代码实现

Class和ClassLoader使用的都是ClassLoader的方法去加载资源

但Class使用resolveName对传入的参数做了特殊处理，从而导致了最终结果的不一致

Class对填写的路径做了解析处理

  • 对于以/开头的值去除/
  • 对于不以/开头的值会添加当前类所在包的包名前缀

总结

Class加载资源和ClassLoader加载资源调的都是ClassLoader的getResource/getResourceAsStream方法

Class加载资源会对填写的路径做特殊处理，/开头去除/，否则添加类所在包的包名前缀

Class加载资源

  • 填写的是相对路径：得到classpath路径 + 类所在的包路径 + 填写的路径

  • 填写的是绝对路径：得到classpath路径 + 填写的路径

ClassLoader加载资源

  • 填写的是相对路径：得到classpath路径 + 填写的路径
  • 填写的是绝对路径：null

mysql group by时不group by null

发表于 2019-11-07

MySQL取消对值为NULL的字段的分组

假定我们现在有一张score表

我们对score表根据grade字段进行分组

可以看到grade字段为NULL的所有数据都被归为一组了

如果这个时候想让grade字段为NULL的数据成为单独的一组，可以使用IFNULL搭配UUID实现

  • IFNULL函数会判断第一个参数是否为NULL，如果为NULL返回第二个参数的值，否则返回第一个参数的值
  • UUID函数会生成一串唯一不重复的UUID

UUID函数生成了唯一不重复的值通过IFNULL函数替代了原本的NULL进行分组

所以分组后的grade字段为NULL的数据都可以单独归为一组

1055错误

当进行分组时可能会发生1055错误

问题原因

这是因为配置了sql_mode=only_full_group_by

这样对于GROUP BY聚合操作，如果SELECT中的列没有出现在GROUP BY中，则这个SQL就是不合法的

查看配置

不同的mysql版本的sql_mode默认配置不同，可以通过以下命令查看配置

  • SELECT @@global.sql_mode;

  • SHOW VARIABLES LIKE 'sql_mode';

MySQL5.6默认没有配置only_full_group_by

MySQL5.7默认配置了only_full_group_by

修改配置

修改MySQL的配置文件，在[mysqld]下配置sql_mode时取消ONLY_FULL_GROUP_BY配置即可

@Accessors

通过该注解可以控制getter、setter方法生成的格式，需要配合@Data或@Getter、@Setter使用

可以在类或成员变量上使用，且在类上使用将应用于所有的成员变量

@Accessors(fluent = true)

生成的getter、setter方法的方法名为属性名，且setter方法返回当前对象

查看反编译后的代码

@Accessors(chain = true)

生成的getter、setter方法名为带get、set前缀的方法名，且setter方法返回当前对象

chain默认为false，但当设置了fluent=true且未设置chain的值时，chain会默认为true

@Accessors(prefix = “a”)

生成的getter、setter方法名为带get、set前缀的方法名，但方法名不包含a前缀
